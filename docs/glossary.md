# Глоссарий проекта

Этот глоссарий фиксирует единый словарь для архитектуры, разработки и эксплуатации локального распределённого чат-сервера.

## 1) Архитектура и узлы

- **Главный сервер (main node)**: Raspberry Pi 5, единственный основной сервер системы.
- **Внутренняя сборка**: физический блок с Raspberry Pi, M5Tab (deprecated в v1.00.00), встроенным M5Cardputer (deprecated в v1.00.00), ESP32-S3 (deprecated в v1.00.00), M5Stamp S3 (deprecated в v1.00.00), Atom S3 (deprecated в v1.00.00) и служебной периферией.
- **Внешние устройства**: отдельные клиентские узлы вне внутренней сборки (M5Cardputer Adv, внешний M5Cardputer, M5StickC Plus 2, T-Embed, Flipper Zero).
- **Внутренний сервисный контроллер**: контроллер внутри корпуса, который разгружает Raspberry Pi или выполняет изолированную служебную роль.
- **Вспомогательный узел**: не главный сервер, а компонент, который расширяет функции или повышает устойчивость.
- **Edge-узел**: ограниченный локальный сетевой узел в отдельной Wi-Fi сети, поднимаемый по команде администратора.
- **Дополнительная локальная развёртка**: запуск edge-профиля на Stamp S3 или ESP32-S3, без замены роли Raspberry Pi.
- **Профиль узла**: чётко ограниченный набор функций, который разрешён конкретному устройству.
- **Карта модулей**: актуальная таблица фактически обнаруженных модулей, их статуса и ролей.

## 2) Шины, транспорт и связь

- **Внутренний I2C-контур**: короткая внутренняя шина для лёгкой служебной передачи (статусы, heartbeat, команды, телеметрия). (deprecated в v1.00.00)
- **Короткая линия I2C**: безопасная длина и схема подключения внутри корпуса, без удалённого растягивания шины.
- **Прямой сервисный канал**: отдельный физический канал (обычно USB serial/Type-C) для служебной связи.
- **Wi-Fi AP**: основная точка доступа, которую поднимает Raspberry Pi.
- **ESP-NOW/BLE/USB serial**: альтернативные транспорты для реалистичной связи с внешними и распределёнными узлами.
- **Тяжёлая нагрузка**: данные, которые нельзя планировать через I2C (медиа, большие логи, файлы, потоковые данные).

## 3) Режимы доступа и роли пользователей

- **Закрытый режим**: доступ через логин или подачу заявки; свободная регистрация отключена.
- **Открытый режим**: доступ через логин, регистрацию и гостевой веб-вход.
- **Гостевой вход**: веб-доступ без полноценного аккаунта, допустим только в открытом режиме.
- **Локальный аккаунт**: учётная запись в локальной базе сервера.
- **Роль guest**: ограниченная веб-роль без права аппаратного входа и создания чатов.
- **Роль user**: обычный пользователь с базовыми правами на чат, блог и поддержку.
- **Роль admin**: административная роль с расширенным доступом к управлению системой.
- **Бан**: запрет доступа аккаунта на неопределённый срок или до снятия ограничения.
- **Временная блокировка**: ограничение доступа с причиной и временем окончания.
- **Чёрный список устройства**: список device-фингерпринтов/идентификаторов с запретом доступа.

## 4) Режимы развёртывания и доступность модулей

- **Основной слой развёртывания**: базовый рабочий слой сервера на Raspberry Pi, который должен быть активен всегда.
- **Дополнительный слой развёртывания**: управляемые edge-узлы в дополнительных локальных сетях.
- **Лимит дополнительных узлов**: максимум одновременно разрешённых edge-узлов, зависящий от фактически подключённых Stamp S3 и ESP32-S3.
- **Hardware discovery**: обнаружение и верификация реального состава модулей на старте и в работе.
- **Hard-critical модуль**: модуль, без которого выбранный профиль системы не должен запускаться.
- **Soft-critical модуль**: модуль, без которого система работает с частичной деградацией.
- **Optional модуль**: модуль, отсутствие которого снижает удобство, но не ломает базовый запуск.
- **Hold-state (безопасный режим ожидания)**: режим при отсутствии hard-critical модуля без аварийного spam-логирования.
- **Пересчёт возможностей**: динамическое ограничение функций на основе текущей карты модулей.

## 5) Данные, протокол и синхронизация

- **Wire-contract**: согласованный формат обмена между сервером и устройствами.
- **Сообщение протокола**: структурированный пакет с типом, метаданными и полезной нагрузкой.
- **Heartbeat**: периодический сигнал живости узла.
- **Telemetry**: служебные метрики состояния узла и системы.
- **Registration flow**: процедура регистрации устройства или клиента в системе.
- **Event log**: журнал событий для последующей обработки и синхронизации.
- **Локальная очередь событий**: буфер событий на узле при временной недоступности сервера.
- **Идемпотентность**: повторная обработка одного и того же события не меняет результат после первого успешного применения.
- **Conflict resolution rule**: формальное правило разрешения конфликтующих обновлений при синхронизации.
- **SQLite (MVP-база)**: основное хранилище сущностей и состояний на этапе MVP.
- **Файловое хранилище**: каталог для медиа, аватаров, загрузок, backup и журналов.

## 6) Безопасность и эксплуатация

- **Defensive security**: защитные, законные меры безопасности без offensive-сценариев.
- **Rate limiting**: ограничение частоты запросов для защиты от перегрузки и злоупотреблений.
- **Brute-force guard**: защита от перебора учётных данных.
- **Изоляция сессий**: разделение и контроль пользовательских сессий без взаимного влияния.
- **Security log**: журнал событий безопасности для аудита и расследования.
- **Incident log**: отдельный журнал инцидентов и защитных срабатываний.
- **Defensive honeypot**: приманочные маршруты и маркеры для обнаружения подозрительной активности.
- **Safe shutdown**: последовательность безопасного завершения с сохранением целостности БД и файловой системы.
- **Degraded mode**: ограниченный режим работы под атакой или при частичной отказе модулей.
- **Backup flow**: процесс резервного копирования и восстановления данных.

## 7) Интерфейс и устройство-специфика

- **M5Tab HMI-панель**: локальный административный интерфейс, который рисует UI сам и не работает как видеотерминал Raspberry Pi. (deprecated в v1.00.00)
- **Вкладка "Сведения"**: экран M5Tab с системными метриками, картой модулей и лимитами развёрток.
- **Вкладка "Админ-панель"**: экран M5Tab для управления пользователями, контентом, RFID и режимами.
- **Вкладка "Развёртывание"**: экран M5Tab для запуска/остановки дополнительных локальных узлов.
- **Комбинация действий устройства**: локальная последовательность кнопок/клавиш/энкодера для быстрого входа после первичной авторизации.
- **Привязка комбинации к устройству**: правило, по которому комбинация работает только на конкретном устройстве.
- **RFID-карта доступа**: карта из централизованного списка, используемая для разрешённого сервисного действия. (PN532 deprecated в v1.00.00)

## 8) Производительность и реалистичность

- **Неположительная роль узла запрещена**: внутренний узел не должен ухудшать систему без полезной функции.
- **Неотрицательный системный вклад**: узел либо разгружает Raspberry Pi, либо добавляет отказоустойчивость/функциональность.
- **Ресурсная реалистичность**: функции допускаются только если устройство тянет их по CPU, памяти, питанию, радио и UX.
- **Ограниченная автономность**: узлы без накопителя не получают фиктивный глубокий офлайн-профиль.
- **Честный MVP-профиль**: минимальный рабочий набор функций без неподтверждённых обещаний.
