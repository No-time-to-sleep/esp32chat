# Матрица устройств

Документ фиксирует реалистичные границы устройств для этапа `v1.00.00`.

> **Примечание:** Устройства, помеченные †, исключены из активной архитектуры.
Цель: явно определить экран, память, накопитель, транспорт и ограничения каждого узла.

## 1) Принятые обозначения

- **Память (класс)**: ориентировочный уровень ресурсов, потому что точные значения могут отличаться по ревизиям.
- **Накопитель**: встроенный или съёмный носитель, доступный для офлайн-данных и медиа.
- **Транспорт**: рекомендуемый канал связи с сервером или соседними узлами.

## 3) Нормализация автономности

- **Базово неавтономные**: M5StickC Plus 2, Flipper Zero (без подтверждённого расширенного профиля), M5Stamp S3, Atom S3.
- **Условно расширяемые**: M5Cardputer, M5Cardputer Adv, T-Embed — только при подтверждённом накопителе и ресурсах.
- **Всегда главный автономный узел**: Raspberry Pi 5.

## 4) Ограничения по транспорту

- I2C только для короткого внутреннего контура и лёгких служебных сообщений.
- Тяжёлые данные (медиа, файлы, большие логи, поток) через I2C не планируются.
- Для тяжёлого внутриблочного обмена использовать UART/USB serial или Wi-Fi по профилю устройства.
- Для внешних устройств используются Wi-Fi/BLE/ESP-NOW/USB serial по профилю устройства.

## 5) Что ещё нужно подтвердить на реальном железе

- Точные объёмы RAM/flash по каждой ревизии плат.
- Наличие и тип съёмного накопителя на конкретных экземплярах устройств.
- Практически доступный аудиоконтур там, где планируются голосовые функции.
