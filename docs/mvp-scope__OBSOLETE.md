# MVP Scope

> **v1.00.00:** С архитектурой v1.00.00 проект перешёл на RPi-Only. Внутренние контроллеры исключены.

Этот документ фиксирует честные границы MVP для проекта локального распределённого чат-сервера.

## 1) Цель MVP

Собрать рабочую, реалистичную систему на базе Raspberry Pi 5 с веб-доступом, базовыми пользовательскими сценариями и ограниченными аппаратными клиентами без неподтверждённых обещаний.

## 2) Что входит в MVP (обязательно)

### Сервер и платформа

- Raspberry Pi 5 как единственный главный сервер и основной Wi-Fi AP.
- Базовый backend (API + веб-интерфейс + хранение данных).
- SQLite как MVP-БД, файловое хранилище для аватаров/вложений/логов.
- Безопасное выключение и базовые журналы безопасности.

### Режимы доступа

- Закрытый режим: вход + подача заявки, без свободной регистрации.
- Открытый режим: вход + регистрация + гостевой вход только для web.
- Роли: guest/user/admin с базовыми ограничениями.

### Веб-интерфейс

- Вкладки: чат, блог, поддержка, аккаунт, устройства, об авторах.
- Чат: список чатов, история, отправка текста, приватные комнаты, лимиты.
- Блог: публикация админом и чтение пользователями.
- Поддержка: обращения, очередь, статусы, ответы администратора.

### Админ-функции

- Управление пользователями (бан/разбан/временная блокировка/blacklist).
- Переключение открытого и закрытого режима безопасным способом.
- Базовое управление RFID-картами через PN532 (без завышенных крипто-обещаний).

### Аппаратные клиенты (базовый профиль)

- Встроенный M5Cardputer: текстовый чат + чтение блога.
- M5Tab: административная HMI-панель с структурированными данными, без видеозеркалирования Raspberry Pi.
- Внешние устройства: только реалистичный текстовый профиль, если аппаратные ограничения не позволяют большего.

### Устойчивость и синхронизация

- Базовый protocol contract server/firmware.
- Локальная очередь событий и идемпотентная синхронизация для ограниченных сценариев.
- Пересчёт доступных функций на основе фактического состава модулей.

## 3) Что не входит в MVP (запрещено/отложено)

- Offensive-сценарии безопасности, doxxing, незаконная деанонимизация.
- Заявления о полной защите от Wi-Fi deauth на уровне невозможных гарантий.
- Полноценная медиа-платформа на устройствах без подтверждённого накопителя/ресурсов.
- Полноценная автономность мини-сервера на узлах без хранилища.
- Передача тяжёлых данных через I2C (видео, большие файлы, большие логи).
- Подмена роли Raspberry Pi вспомогательными контроллерами в штатном режиме.

## 4) Условно входит в MVP (только при подтверждённом железе)

- Файлы/фото на клиентском устройстве только при наличии реального накопителя и ресурсов.
- Голосовые сообщения только при доступном аудиоконтуре и стабильной реализации.
- Расширенный автономный профиль только при наличии хранилища и понятной sync-модели.

## 5) Реалистичные ограничения по устройствам

- M5StickC Plus 2 и Flipper Zero в базовой конфигурации считаются неавтономными клиентами.
- T-Embed, M5Cardputer, M5Cardputer Adv получают расширения только при аппаратном обосновании.
- Дополнительные локальные edge-узлы поднимаются только на 3x M5Stamp S3 и 1x ESP32-S3, в пределах фактической доступности.

## 6) Базовые критерии соответствия MVP

- Нет ложных обещаний по железу, автономности, медиа и безопасности.
- Все обязательные пользовательские сценарии web работают в локальной сети.
- Главный сервер стабилен и не перегружается в целевом MVP-диапазоне.
- Неподдерживаемые сценарии явно помечены как ограничения.
