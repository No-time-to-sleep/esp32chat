# Project Roadmap

Этот файл хранит полный roadmap проекта.
Для активной работы и восстановления контекста использовать в таком порядке:

- [SESSION_CONTEXT.md](/D:/project/SESSION_CONTEXT.md) как постоянный блок правил;
- [STAGE_CONTEXT.md](/D:/project/STAGE_CONTEXT.md) как блок текущего этапа;
- [SUBSTAGE_CONTEXT.md](/D:/project/SUBSTAGE_CONTEXT.md) как блок текущего подэтапа.

`PLAN.md` нужен как полный источник дорожной карты и точечной сверки, а не как единственный активный контекст.

  - v1.00.00 RPi-Only рефакторинг: контекст и документация обновлены под RPi-Only; делаем — чистка документации, пометка внутренних прошивок deprecated, RFID/NFC помечен как deprecated, настройка systemd/log rotation/boot self-test, Level 2 тестирование всех устройств, баг-репорт по
    найденным проблемам; результат — проект полностью переориентирован на RPi-Only архитектуру; проверка — ни одна internal-прошивка не упоминается как актуальная, все RFID-ссылки снабжены пометкой deprecated; далее — v1.00.01.
  - v1.00.01 Стабилизация сервера (по результатам тестов): исправление багов из Level 2 тестирования; делаем — фиксы критических и высоких приоритетов, улучшение обработки ошибок, оптимизация SQLite (WAL, auto-vacuum); результат — сервер проходит повторное Level 2 без
    ошибок; проверка — все баги из репорта имеют статус fixed или acknowledged; далее — v1.00.02.
  - v1.00.02 Долгая стабильность: тестирование 2+ недели uptime; делаем — Pi Connect / headless setup, health monitoring dashboard; результат — сервер стабильно работает 14+ дней без деградации; проверка — метрики dashboard не показывают утечек памяти или роста времени
    ответа; далее — v1.00.03.
  - v1.00.03 RC1 RPi-Only: финальная стабилизация; делаем — релиз-кандидат для развёртывания на Pi, документация по эксплуатации; результат — готовый RC1 образ для Raspberry Pi; проверка — deploy по инструкции приводит к работающему серверу с корректным логированием и
    автостартом; далее — следующий цикл по багфиксам.
